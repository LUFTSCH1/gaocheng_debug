使用说明

简单上手

  • 新项目
    • 点击 新建 ，在输入对应题号后新建一个项目
    • 点击对应的 选择 以选择官方demo路径（在弹出的对话框中找到demo的exe文件，双击或选中后点击 打开 ）
    • 选择作业exe路径，操作同上
      PS: 点击 设置 可以更改对话框默认浏览目录，更加方便
    • 选择--trim参数、--display参数
    • 点击 创建/修改测试数据 ，进行数据添加：
      本应用内测试数据序号支持简便输入（即中括号仅输左边）与常规输入，
      其余合法格式如下图（行首"["后的同行字符会被忽略，详见细节）

      若要测试的程序无输入，可以空置或输入一个'['，但一定要完成 创建/修改测试数据 操作（原因见细节）
    • 完成数据编辑后点击 生成/修改并开始测试
    • 若想查看生成文件，可点击 打开项目目录 ，即可查看对应文件
  • 现有项目
    • 点击下拉框选择一个已有的项目
    • 基本操作同新项目，若仅修改--trim或--display参数或是exe路径则可直接点击 重复测试

细节

  • 点击 新建 后，会新建一个项目目录，且会包含以下文件：
    • _project.gaocheng
    点击 创建/修改测试数据 窗口的 生成/修改并开始测试 后，会进行如下操作：
    • 关闭数据生成/修改窗口
    • 新建/修改测试数据文档（_test_data.txt）
    • 修改_project.gaocheng
    • 在CMD中运行测试
    • 将测试结果显示在文本框中
    点击 重复测试 后，会进行如下操作：
    • 必要时修改_project.gaocheng
    • 在CMD中运行测试
    • 将测试结果显示在文本框中
  • 创建/修改数据时会对简写的数据自动编号（不超过256个）
  • 测试数据序号自动生成会将行首'['到换行符为止替换为对应合法序号
  • 看起来多余的 重复测试 按钮是为了方便需要同一数据多次测试的情况，例如：
    在测试数据不变的情况下 希望更改--trim或--display参数 / 希望更改demo或作业exe路径
    （常见用于测试完C++实现后测试C实现）
    程序内有输出运行时间 / 输出有不可预测值 等

常规

  • 您的测试数据中不应包含任何"["（其之后整行会被替换为测试数据编号）
  • 您的测试数据中不应包含任何";" "//" 和 "#"（会被get_input_data.exe当做注释）
  • 应用修改测试数据时会自动重新编号，因此您无需担心插入数据会造成编号重复
  • demo和您的作业exe在测试的CMD窗口关闭前会被保护，防止中途更改
  • 由于本应用开启的CMD标准输入流被重定向到本应用内部，用户无法使用键盘进行输入
  • 本应用生成的所有文件均在test_log目录下，要卸载本应用直接将程序所处的目录（默认为Debug_Tool）删除即可
  • 本应用在Windows下运行，使用本应用可能需要安装.NET运行环境
  • 不建议对rsc目录手动进行任何更改
